# User Personas - AWSomeShop

## Persona 1: 员工 (Employee)

### 基本信息
- **角色**: 公司员工
- **技术水平**: 基础到中等（能够使用常规网站）
- **使用频率**: 每月 2-4 次（查看积分、兑换产品）

### 目标
- 快速浏览可兑换的产品
- 使用积分兑换心仪的产品
- 了解自己的积分余额和历史
- 查看过往兑换记录

### 痛点
- 不清楚自己有多少积分可用
- 不知道积分是如何获得的
- 担心兑换失败或积分扣除错误
- 希望快速找到想要的产品

### 使用场景
1. **定期查看**: 每月登录查看新增积分和可兑换产品
2. **积分兑换**: 发现喜欢的产品后立即兑换
3. **历史查询**: 查看过往兑换记录和积分变动

### 期望
- 界面简洁，操作直观
- 积分信息清晰可见
- 兑换流程简单快捷
- 能够追溯积分来源

## Persona 2: 管理员 (Admin)

### 基本信息
- **角色**: HR 或福利管理人员
- **技术水平**: 中等到高级（熟悉后台管理系统）
- **使用频率**: 每周 3-5 次（管理产品和积分）

### 目标
- 高效管理产品库（添加、编辑、下架）
- 灵活管理员工积分（发放、扣除）
- 监控系统使用情况
- 确保积分系统公平运行

### 痛点
- 手动管理大量员工积分耗时
- 需要快速响应员工的积分问题
- 产品信息更新频繁
- 需要追踪积分操作历史

### 使用场景
1. **产品管理**: 定期更新产品信息，上架新产品
2. **积分发放**: 每月为员工发放固定积分
3. **积分调整**: 处理特殊情况（奖励、纠错）
4. **数据查询**: 查看员工积分状态和系统使用情况

### 期望
- 批量操作功能（批量发放积分）
- 清晰的操作日志
- 简单的产品管理界面
- 快速查询员工积分信息
